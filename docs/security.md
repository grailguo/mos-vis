# MOS-VIS 安全与合规设计（Security Spec）

来源：`SystemDesgin.md` 专题拆分  
范围：传输安全、访问控制、隐私、合规、供应链

## 1. 安全目标
- 防止控制通道未授权访问。
- 防止语音与配置数据泄露。
- 满足嵌入式产品的可审计与可升级安全要求。

## 2. 传输与认证

### 2.1 WebSocket 安全
- 生产环境启用 `WSS(TLS)`。
- 明文 `WS` 仅允许开发环境。

配置项：
- `control.wss_enabled`
- `control.tls_cert_path`
- `control.tls_key_path`
- `control.auth_token`

### 2.2 首包认证
```json
{"type":"auth","token":"<pre-shared-token>"}
```

未认证请求处理：
- 返回 `401`。
- 写入审计日志（来源、时间、动作）。

### 2.3 敏感操作授权
敏感动作（例如恢复出厂设置、修改唤醒词）需满足至少一项：
- 本地 Unix Socket 调用。
- 远程认证 + 授权码校验。

## 3. 数据安全与生命周期

### 3.1 音频数据
- 会话结束后环形缓冲区数据可清零。
- 默认不持久化原始音频。

### 3.2 文本与偏好数据
- ASR 结果默认不持久化。
- 用户偏好与调试数据可选 AES-256 加密。
- 密钥建议托管于受保护分区或硬件安全模块。

### 3.3 调试上传
- 默认关闭。
- 启用前需显式用户同意。
- 提供日志清除 API。

## 4. 错误与审计
统一错误码：
- `200` 成功
- `400` 请求格式错误
- `401` 认证失败
- `404` 动作不存在
- `500` 内部错误
- `503` 设备繁忙

审计建议字段：
- `request_id`、`client_id`、`action`、`result_code`、`timestamp`

## 5. OTA 安全
- 升级包必须签名验证。
- 验签失败立即终止升级。
- 关键组件升级失败自动回滚。

## 6. 法规与许可证

### 6.1 隐私法规
- 中国：遵循《个人信息保护法》。
- 海外：按区域执行 GDPR/CCPA。

### 6.2 无线与设备认证
- 如产品含 Wi-Fi/BT，上市前需完成 FCC/CE/RED 认证。

### 6.3 开源许可证
- 维护第三方依赖许可证清单。
- 发布包包含 `NOTICE` 文件。

## 7. 供应链安全（路线图）
- 建立关键器件替代清单（一级/二级）。
- 跟踪核心器件 EOL 与长期供货风险。
- 固件发布链路引入签名与来源追踪。

## 8. 里程碑
- P0：WSS + 认证 + 错误码 + 隐私默认策略。
- P1：敏感操作细粒度授权 + 审计完善。
- P2：供应链安全与认证体系扩展。
